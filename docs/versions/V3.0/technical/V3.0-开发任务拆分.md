## 四、剩余部署与联调任务

### 4.1 云部署配置

需要你在云部署侧完成：

| 任务 | 说明 |
|------|------|
| 部署 `play-web-service` | 将当前服务部署到云部署 |
| 按需覆盖环境变量 | 仅在需要切环境或收紧跨域时覆盖默认值 |
| 确认公网域名 | 用于生成最终 NFC 播放页链接 |

`Dockerfile` 默认内置：

- `CLOUD_ENV_ID`
- `CORS_ALLOW_ORIGIN=*`

按需在控制台覆盖：

- `CLOUD_ENV_ID`
- `CORS_ALLOW_ORIGIN`

说明：
- 当前主线为云部署容器方案，以 `Dockerfile` 和服务环境变量为准
- 微信云开发云部署当前主线使用运行环境自带身份访问 CloudBase
- 当前代码不再显式传入 `secretId / secretKey / sessionToken`
- 当前主线默认只保存 `fileId`，由云部署后端在访问时换取媒体播放地址
- 云存储为私有读时，浏览器每次访问播放页都会重新获取可播放地址
- 云存储为公有读时，可直接获得长期可访问的媒体地址
- 即使采用公有读，当前主线仍以 `fileId` 作为唯一文件标识，并由后端统一换取后返回给网页

### 4.2 小程序回填

部署完成后需要回填：

- `miniprogram/utils/cloud-config.js`
  - `PLAY_BASE_URL`

### 4.3 联调验证

需要完成：

1. 上传一条真实音频
2. 上传一条真实视频
3. 分别生成 `pixel / minimal / poster` 分享链接
4. 浏览器访问三种风格页面
5. 验证音频播放、视频播放
6. 删除文件后再次访问旧链接，确认提示“链接不存在或已删除”

## 六、完成标准

V3.0 可视为“开发完成”的最小标准：

- 小程序端可上传音频和视频
- 标题可选，风格可选
- NFC 写入的是固定播放页链接
- 浏览器端可通过云部署后端读取分享详情
- 音频、视频都可播放
- 三种风格全部可访问
- 删除文件后旧链接提示“链接不存在或已删除”
- 文档、原型、UI 与当前实现保持一致
