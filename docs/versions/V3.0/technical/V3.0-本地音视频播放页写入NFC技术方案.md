## 一、方案概述
V3.0 采用“播放页链接写入 NFC”的方案：

1. 小程序选择本地音频或视频文件  
2. 小程序校验格式、大小、标题和风格  
3. 文件上传到云存储  
4. 云函数创建 `media_files` 文件记录  
5. 云函数创建 `media_shares` 分享记录并生成 `shareId`  
6. 小程序生成 `/play/{themeKey}/{shareId}` 播放页链接  
7. NFC 写入播放页链接  
8. 浏览器打开 `/play/{themeKey}`  
9. 云部署服务查询分享记录和文件记录  
10. 云部署服务生成媒体播放地址  
11. 网页端渲染音频或视频播放器

当前网页端统一落在：

```text
play-web-service/
```

## 三、配置项
### 3.1 小程序端配置
文件：

```text
miniprogram/utils/cloud-config.js
```

当前关键常量：

```js
const CLOUD_ENV_ID = 'cloud1-5gsbvhr91440e872'
const MEDIA_SHARE_SERVICE_NAME = 'media-share-service'
const PLAY_BASE_URL = ''
```

说明：
- `PLAY_BASE_URL` 用于生成 NFC 写入的播放页链接
- 发布前必须替换为真实云部署域名
- 如果保持空字符串，代码会视为“未配置”，阻止生成正式链接

### 3.2 云部署环境变量
当前默认值已写入 `Dockerfile`：

| 变量名 | 说明 |
|--------|------|
| `CLOUD_ENV_ID` | 云开发环境 ID |
| `CORS_ALLOW_ORIGIN` | `cors` 中间件允许访问的站点来源 |

补充说明：
- 当前主线使用云部署容器方案，以 `Dockerfile` 作为启动与默认环境配置入口
- 默认容器端口为 `3000`，可通过云部署服务配置覆盖 `PORT`
- `Dockerfile` 已内置：
  - `NODE_ENV=production`
  - `PORT=3000`
  - `CLOUD_ENV_ID=cloud1-5gsbvhr91440e872`
  - `CORS_ALLOW_ORIGIN=*`
- 如需切换环境或收紧跨域来源，可在云部署控制台中覆盖默认值
- 微信云开发云部署场景下，优先使用运行环境自带身份访问 CloudBase
- 当前主线代码不再显式传入 `secretId / secretKey / sessionToken`
- 当前主线以 `fileId` 为准，由云部署后端在访问时换取媒体播放地址
- 若云存储为私有读，浏览器每次打开播放页时重新获取当次可播放地址
- 若云存储为公有读，则可直接获得长期可访问的媒体地址
- 如后续迁移到非云开发托管环境，再单独补充密钥初始化方案

## 四、数据库设计
### 4.1 `media_files`
| 字段 | 类型 | 说明 |
|------|------|------|
| `openid` | String | 文件所属用户 |
| `mediaType` | String | `audio / video` |
| `fileId` | String | 云存储文件 ID |
| `fileName` | String | 原始文件名 |
| `fileExt` | String | 文件扩展名 |
| `fileSize` | Number | 文件大小 |
| `createdAt` | Number | 创建时间戳 |
| `updatedAt` | Number | 更新时间戳 |

### 4.2 `media_shares`
| 字段 | 类型 | 说明 |
|------|------|------|
| `shareId` | String | 不可猜测的随机分享 ID |
| `fileRecordId` | String | 关联 `media_files` 记录 |
| `openid` | String | 分享所属用户 |
| `mediaType` | String | `audio / video` |
| `themeKey` | String | `pixel / minimal / poster` |
| `title` | String | 播放页标题，可为空 |
| `createdAt` | Number | 创建时间戳 |
| `updatedAt` | Number | 更新时间戳 |

说明：
- 删除采用真删除，不保留 `status`、`deletedAt`
- 同一个文件可以生成多条分享记录

## 五、小程序端实现
### 5.1 文件选择
- 使用 `wx.chooseMessageFile`
- 支持 `mp3 / m4a / wav / mp4`
- 音频大小上限 `10MB`
- 视频大小上限 `20MB`

### 5.2 文件上传
- 通过 `wx.cloud.uploadFile` 上传到云存储
- 路径由 `buildCloudMediaPath()` 生成：
  - `media-files/audio/...`
  - `media-files/video/...`

### 5.3 链接生成
通过：

```js
buildPlayPageUrl(shareId, themeKey)
```

生成链接：

```text
https://你的云部署域名/play/pixel/shareId
https://你的云部署域名/play/minimal/shareId
https://你的云部署域名/play/poster/shareId
```

补充规则：
- 播放页正式路由固定为 `/play/{themeKey}/{shareId}`
- 如果 `PLAY_BASE_URL` 未配置，则阻止生成正式链接

### 5.4 标题规则
- 标题可选
- 不填标题也允许创建分享记录
- 如果标题为空，网页端隐藏标题区

## 六、云函数实现
### 6.1 `createFile`
职责：
- 校验文件标识、文件名、扩展名、媒体类型、大小
- 创建 `media_files` 记录

### 6.2 `createShare`
职责：
- 校验文件记录是否存在
- 校验风格是否合法
- 生成唯一 `shareId`
- 创建 `media_shares` 记录

### 6.3 `listFiles`
职责：
- 查询当前用户文件列表
- 汇总每个文件的最近分享信息
- 返回最近一次分享的 `shareId / title / themeKey`

### 6.4 `deleteFile`
职责：
- 删除云存储文件
- 删除该文件关联的分享记录
- 删除文件记录

删除顺序：
1. 删除云存储文件  
2. 删除 `media_shares`  
3. 删除 `media_files`

如果云存储删除失败：
- 不继续删除数据库记录
- 直接返回失败

## 七、云部署服务实现
### 7.1 当前接口
`play-web-service` 当前提供：

| 路径 | 说明 |
|------|------|
| `/health` | 健康检查 |
| `/api/share/detail?id=shareId` | 查询分享详情 |
| `/play/pixel/shareId` | Pixel 风格播放页 |
| `/play/minimal/shareId` | Minimal 风格播放页 |
| `/play/poster/shareId` | Poster 风格播放页 |

跨域实现说明：
- `app.js` 中通过 `cors` 中间件处理 `GET / POST / OPTIONS`
- 仅 `/api` 与 JSON 错误响应使用 `Cache-Control: no-store`
- 静态官网、播放页 HTML 和静态资源保留缓存，降低带宽消耗
- 不再手写 `Access-Control-Allow-*` 响应头

### 7.2 `/api/share/detail`
后端流程：
1. 读取 `shareId`
2. 查询 `media_shares`
3. 查询关联 `media_files`
4. 调用 CloudBase 获取媒体播放地址
5. 返回前端所需字段

返回示例：

```json
{
  "success": true,
  "share": {
    "shareId": "AbCdEf12345678",
    "mediaType": "audio",
    "themeKey": "pixel",
    "title": "生日祝福",
    "fileName": "voice.m4a",
    "mediaUrl": "https://..."
  }
}
```

错误返回示例：

```json
{
  "success": false,
  "code": "SHARE_NOT_FOUND",
  "message": "链接不存在或已删除"
}
```

错误处理策略：
- 业务可预期错误通过 `HttpError` 抛出，并按原状态码与文案返回
- 未知异常仅在服务端记录日志
- 浏览器侧统一收到通用错误文案，避免暴露 CloudBase 或运行时内部细节

### 7.3 媒体地址生成
后端通过 CloudBase Node SDK：

```js
getTempFileURL()
```

生成 `mediaUrl`。

说明：
- NFC 写入的长期有效地址是 `/play/{themeKey}/{shareId}`
- 当前主线默认只在数据库中保存 `fileId`
- 云部署后端在访问时通过 `getTempFileURL()` 换取当前可播放地址
- 云存储为私有读时，浏览器每次访问播放页都会重新获取可播放地址
- 云存储为公有读时，可直接获得长期可访问的媒体地址
- 即使采用公有读，当前主线仍以 `fileId` 作为唯一文件标识，并由后端统一换取后返回给网页
- 当前主线不再依赖项目内自定义的链接过期时间配置

## 八、播放页实现
### 8.1 目录结构
```text
play-web-service/
  public/
    play/
      pixel/
        index.html
        style.css
        app.js
      minimal/
        index.html
        style.css
        app.js
      poster/
        index.html
        style.css
        app.js
      shared/
        core.js
```

### 8.2 页面规则
- 三种风格独立目录维护
- 音频与视频共用同一风格目录，但按 `mediaType` 切换播放器
- 标题为空时隐藏标题区
- 状态页包括：
  - 加载中
  - 链接不存在或已删除
  - 加载失败
