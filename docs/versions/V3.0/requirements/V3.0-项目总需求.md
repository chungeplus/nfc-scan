# NFC卡片助手需求文档

**所属产品**：NFC卡片助手  
**所属版本**：V3.0（开发中）  
**最后更新**：2026-04-29  
**文档状态**：已按当前代码主线同步

---

## 一、产品概述

NFC卡片助手用于将不同类型的链接写入 NFC 标签，用户碰卡后可快速打开目标内容。

当前代码主线保留了原有三类写入能力：

- 应用写入
- 音乐写入
- 网页写入

并在此基础上新增了本地音视频写入能力：

- 小程序端上传本地音频或视频
- 生成固定播放页链接
- 将播放页链接写入 NFC
- 浏览器打开播放页后进行播放

---

## 二、当前版本定位

### 2.1 当前主线

当前代码主线已经升级为 V3.0 方案，核心特征如下：

- 小程序页面路径统一为 `write-local-media`
- 小程序云函数统一为 `media-share-service`
- 数据集合统一为：
  - `media_files`
  - `media_shares`
- NFC 写入的不是媒体文件直链，而是播放页链接
- 网页播放页由 `play-web-service` 提供

### 2.2 历史版本说明

V2.0 不再保留整套旧需求与旧技术文档，当前仅保留版本归档说明，作为阶段记录，不再代表当前代码结构与实现方式。

## 三、功能范围

### 3.1 保留功能

| 模块 | 说明 |
|------|------|
| 应用专区 | 写入应用链接或唤起信息 |
| 音乐专区 | 写入音乐分享链接 |
| 网页专区 | 写入普通网页链接 |

### 3.2 当前新增能力

| 模块 | 说明 |
|------|------|
| 本地音视频 | 从微信会话选择音频或视频文件，生成播放页链接并写入 NFC |
| 我的文件 | 查看已上传文件、再次生成链接、删除文件 |
| 浏览器播放页 | 浏览器通过播放页链接展示音频或视频播放器 |

## 四、页面与路由

### 4.1 小程序页面

| 页面 | 路径 | 说明 |
|------|------|------|
| NFC 首页 | `pages/write-menu/write-menu` | 原有写入入口聚合页 |
| 我的文件 | `pages/my-files/my-files` | 查看文件、再次写入、删除 |
| 应用专区 | `pages/write-app/write-app` | 原有功能 |
| 音乐专区 | `pages/write-music/write-music` | 原有功能 |
| 网页专区 | `pages/write-web/write-web` | 原有功能 |
| 本地音视频 | `pages/write-local-media/write-local-media` | 上传、建档、选风格、写卡 |

### 4.2 网页端路由

网页端不属于小程序页面栈，由云部署服务统一提供：

| 路径 | 说明 |
|------|------|
| `/play/pixel/shareId` | Pixel 风格播放页 |
| `/play/minimal/shareId` | Minimal 风格播放页 |
| `/play/poster/shareId` | Poster 风格播放页 |
| `/api/share/detail?id=shareId` | 播放页详情接口 |
| `/health` | 服务健康检查 |

## 五、核心流程

### 5.1 本地音视频上传与写卡

```text
用户进入 NFC 首页
    ->
点击“本地音视频”
    ->
从微信会话选择文件
    ->
系统校验格式、大小、标题和风格
    ->
上传到云存储
    ->
云函数创建文件记录
    ->
云函数创建分享记录
    ->
生成 /play/{themeKey}/{shareId}
    ->
写入 NFC
```

### 5.2 我的文件复用

```text
用户进入“我的文件”
    ->
查看媒体文件
    ->
点击“去写入”
    ->
回到本地音视频页
    ->
重新设置标题和风格
    ->
生成新的分享链接
    ->
继续写卡
```

### 5.3 浏览器播放

```text
用户碰卡
    ->
浏览器打开 /play/{themeKey}/{shareId}
    ->
网页端请求 /api/share/detail
    ->
服务端查询 media_shares 和 media_files
    ->
服务端生成临时 mediaUrl
    ->
网页端渲染播放器
```

### 5.4 删除

```text
用户在“我的文件”删除文件
    ->
系统删除云存储文件
    ->
删除 media_shares
    ->
删除 media_files
    ->
旧链接打开播放页时提示“链接不存在或已删除”
```

## 七、当前边界

- 文件选择入口来自微信会话，不承诺系统文件管理器或相册任意选择
- 音频支持：
  - `mp3 / m4a / wav`
  - 不超过 `10MB`
- 视频支持：
  - `mp4`
  - 不超过 `20MB`
- 浏览器播放页不承诺自动播放
- 不提供防下载能力
- 删除策略为真删除，不做软删除
