- 音乐页网络请求失败后卡在"解析中"

- 音乐链接空解析结果继续写入无效 URI
- NFC 弹窗重试时旧状态覆盖新状态
- NFC 弹窗重复发现同一标签可能重复写入
- 网页 URL 校验过严，可能误伤合法链接
- 应用页 iOS 按钮状态与未接入提示不一致
- 开发者承诺弹窗改为仅首次进入首页展示
- 小程序已接入全局版本更新检测

## 4. 待真机验证项

以下项目代码侧已修复，但仍建议在真机上完成最终确认：

### 4.1 音乐写入

- 网易云短链真实跳转是否稳定
- QQ 音乐分享链接真实跳转是否稳定
- 弱网、断网下是否能正确退出"解析中"

### 4.2 网页写入

- 带 `query` 参数的 URL
- 带 `hash` 锚点的 URL
- 带 `port` 端口的 URL
- 无协议输入自动补全 `https://`

### 4.3 应用写入

- Android 下应用选择、包名回填、写入流程
- iOS 下切换提示是否符合预期
- iOS 下写入按钮禁用态是否清晰

### 4.4 NFC 弹窗

- 非 NDEF 标签错误提示
- 写入失败后点击"重试"
- 连续快速贴卡是否重复写入
- 快速开关弹窗是否有状态错乱

## 5. 风险评估

### 5.1 低风险

- 页面结构、样式、交互一致性
- 音乐页基础解析状态流转
- 网页页输入与 URL 规范化
- 应用页 iOS 按钮状态一致性

### 5.2 中风险

- 音乐分享链接真实跳转依赖第三方链接结构
- NFC 真机写入依赖设备系统、标签类型、机型差异
- 重试与重复发现问题虽已修复，仍需真机高频验证

## 6. 发布建议

   - iOS 未接入提示
   - 按钮禁用态
3. 弱网 / 断网验证：
   - 音乐解析失败恢复
4. 特殊标签验证：
   - 非 NDEF
   - 已写入标签
