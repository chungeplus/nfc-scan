## 一、项目概述

### 1.1 项目背景

`NFC卡片助手` 是一款微信小程序，核心能力是将不同类型的内容写入 NFC 标签，实现“碰一碰直达”。

当前版本已支持以下内容写入：

- 应用包名
- 音乐分享链接
- 网页 URL
- 本地音频文件上传后生成的可访问链接

用户使用支持 NFC 的手机靠近标签后，即可触发目标内容的打开或跳转。

### 1.2 目标用户

- 需要管理 NFC 标签的个人或小型商家用户
- 希望快速分享应用、音乐、网页或语音内容的用户
- 对像素风工具类界面有偏好的用户

### 1.3 项目范围

| 包含 | 不包含 |
|------|--------|
| 首页功能入口 | 视频上传与写入 |
| 应用包名写入 | 用户登录/注册体系 |
| 音乐链接解析写入 | 后台管理系统 |
| 网页 URL 写入 | 标签批量管理 |
| 本地音频上传写入 | 独立 H5 播放页方案 |
| 我的文件管理 | 多主题媒体播放方案 |
| NFC 写卡交互反馈 | |

### 1.4 本次版本更新说明

本次更新为增量版本更新，保留原有“应用写入 / 音乐写入 / 网页写入”三类能力，在此基础上新增“本地音频上传写入 NFC”与“我的文件管理”功能。

本次新增内容包括：

- 本地音频文件选择与上传
- 云函数写入音频记录
- 我的文件列表查看
- 已上传音频再次写入 NFC
- 已上传音频删除
- 自定义 `tabBar`（`NFC / 我的文件`）

### 2.2 用户流程

#### 2.2.1 原有功能流程

```text
用户打开小程序
    ->
首次进入 -> 显示开发者承诺弹窗 -> 用户点击“我知道了” -> 关闭弹窗
    ->
NFC 设备检测 -> 不支持 -> 显示提示 -> 流程结束
    -> 支持
显示首页（四个功能入口）
    ->
点击“应用” -> 进入写入应用页 -> 填写包名 -> 点击写入 -> NFC 写卡流程
点击“音乐” -> 进入写入音乐页 -> 粘贴链接 -> 解析 -> 点击写入 -> NFC 写卡流程
点击“网页” -> 进入写入网页页 -> 填写 URL -> 点击写入 -> NFC 写卡流程
点击“本地音频” -> 进入写入本地音频页 -> 执行本地音频流程
```

#### 2.2.2 本地音频写入流程

```text
用户打开小程序
    ->
进入 NFC 首页
    ->
点击“本地音频”
    ->
从微信会话选择音频文件
    ->
校验文件格式与大小
    ->
上传到云存储
    ->
调用云函数写入数据库记录
    ->
返回可写入链接
    ->
打开 NFC 写入弹窗
    ->
用户贴近 NFC 标签
    ->
写入成功
    ->
用户可选择“继续写入”或“回我的文件”
```

#### 2.2.3 我的文件使用流程

```text
用户点击底部“我的文件”
    ->
页面先展示“正在加载我的文件”状态
    ->
查看已上传音频列表
    ->
可选择“去写入”进入本地音频页继续写卡
    ->
若当前记录未获取到可用链接，则“去写入”按钮禁用，需先检查云存储权限并下拉刷新
    ->
或选择“删除”移除当前文件记录
    ->
删除后旧 NFC 链接失效
```

#### 2.2.4 NFC 写卡流程

```text
1. 显示“请将 NFC 标签贴近手机”
2. 用户贴近标签 -> 检测到标签
3. 显示“写入中，请勿移动标签”
4. 写入成功 -> 显示成功提示
5. 写入失败 -> 显示错误原因 -> 支持重试
```

### 2.4 新增功能说明

#### 2.4.1 本地音频上传写入说明

- 当前版本不提供小程序内录音能力，采用“从微信会话选择音频文件”的方式上传
- 当前使用 `wx.chooseMessageFile` 选择音频文件
- 当前支持格式为：`mp3 / m4a / wav`
- 当前文件大小上限为：`20MB`
- 音频文件上传后，系统通过云函数写入数据库记录，并返回可用于写入 NFC 的播放链接
- 页面字段当前显示为“播放页链接”，但当前实际生成的是云存储可访问的播放链接，不是独立 H5 播放页
- 用户写卡成功后，可继续写入其他标签，或返回“我的文件”继续管理

#### 2.4.2 我的文件说明

- 展示当前用户已上传音频文件列表
- 展示文件总数与云端占用
- 页面进入后先展示“正在加载我的文件”状态，提示云函数正在返回当前用户记录与最新可用播放链接
- 当前默认最多展示最近 `50` 条有效记录
- 每条记录展示文件名、上传时间、文件大小、播放链接
- 支持“去写入”，将当前记录带回本地音频写入页继续写卡
- 当记录未取得可用播放链接时，“去写入”按钮禁用，并提示检查云存储 `audio-files` 路径权限
- 支持删除文件记录
- 删除前弹出确认提示
- 删除后，原先已写入 NFC 的旧链接将失效

### 2.5 异常处理

| 场景 | 处理方式 |
|------|----------|
| 手机不支持 NFC | 弹窗提示当前设备不支持 NFC 写入 |
| NFC 写入失败 | 显示错误原因，并提供重试入口 |
| 包名格式错误 | 输入框下方显示格式错误提示 |
| URL 格式错误 | 输入框下方显示“请输入有效的网址” |
| 音乐链接解析失败 | Toast 提示“暂不支持该链接格式” |
| 音频格式不支持 | 提示“仅支持 mp3 / m4a / wav 文件” |
| 音频文件过大 | 提示“文件超过 20MB，请压缩后重新上传” |
| 文件读取失败 | 提示“无法读取文件，请重新选择” |
| 上传失败 | 提示“上传失败，请稍后重试” |
| 链接不可用 | 提示检查云存储 `audio-files` 路径权限 |
| 删除失败 | 提示“删除失败，请稍后重试” |
| 小程序存在新版本 | 弹出更新提示，用户确认后重启更新 |

## 三、UI/UX 设计规范

### 3.1 整体风格

**风格名称**：纸张底像素风 8-bit 工具型 UI

**设计原则**：

- 页面主体使用暖纸色背景，外层场景色使用浅绿色承托
- 核心组件保留粗黑描边、硬阴影、像素切角和块面层级
- 首页、表单页、弹窗页统一使用“纸张底 + 彩色功能块 + 黑色描边 + 硬阴影”
- 自定义 `tabBar` 需与现有像素风视觉一致

### 3.2 颜色规范

| 类别 | 色值 | 用途 |
|------|------|------|
| 场景绿 | `#88AA67` | 页面外层背景 |
| 纸张底 | `#F4EDD6` | 页面主体背景 |
| 纸张辅底 | `#EFE5C6` | 输入框、浅色容器、次按钮背景 |
| 主描边 | `#2D2D2D` | 全局边框、图标主线 |
| 次文字 | `#626262` | 描述、说明、辅助信息 |
| 橙色 | `#F0BF66` | 首页应用卡片 |
| 紫色 | `#CEB0EF` | 音乐卡片 |
| 青色 | `#9FD9DF` | 网页卡片 |
| 绿色 | `#B8D98B` | 成功态、确认态 |
| 深绿 | `#6EA85E` | 主按钮 |
| 红色 | `#E86A58` | 失败态、警告态 |
| 深红 | `#BE4A3A` | 错误文字 |
| 黄色 | `#EFD86C` | 标题条、次按钮、强调说明 |

### 3.3 关键界面要求

- 首页展示 4 个入口卡片
- 底部自定义 `tabBar` 文案为 `NFC / 我的文件`
- `写入本地音频` 页面右上角不显示 Home 按钮
- 客服悬浮按钮不得被底部 `tabBar` 遮挡
- 删除确认弹窗需明确提示“旧 NFC 链接失效”
