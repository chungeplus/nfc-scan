## 一、功能概述

### 1.1 需求背景

用户希望能够将**自己的录音**写入NFC标签，实现"碰一碰播放录音"的功能。相比于音乐链接写入，这个功能允许用户自定义录音内容，适用于：

- 个人语音留言/留言板
- 产品说明/景点介绍（商家自定义）
- 语音名片/个人介绍
- 教育场景的语音指导

### 1.2 核心价值

| 传统方案 | 本功能方案 |
|----------|------------|
| 快捷指令跳转（仅自己手机可用） | 任何NFC手机碰一碰都能播放 |
| 只能跳转到录音App | 直接打开H5播放页面 |
| 需要安装特定App | 浏览器直接打开 |
| 录音只能自己听 | 可以分享给任何人 |

### 1.3 技术选型

| 组件 | 技术方案 | 说明 |
|------|----------|------|
| 录音存储 | 微信云开发云存储 | 支持永久访问URL，配置权限后外部可访问 |
| H5播放页 | 云托管静态网站 | 支持HTTPS外部访问，无需服务器 |
| NFC写入 | 微信小程序NFC API | 写入H5页面URL到NFC标签 |

## 四、技术方案

### 4.1 云存储配置

**存储权限设置**：
- 将录音存储目录权限设置为**"所有用户可读"**
- 确保外部浏览器可以直接访问录音文件

**文件上传代码示例**：

```javascript
// 微信小程序端上传录音
wx.cloud.uploadFile({
  cloudPath: `recordings/${Date.now()}.m4a`,
  filePath: tempFilePath, // 录音临时文件路径
  success: res => {
    // 获取永久访问URL
    wx.cloud.getTempFileURL({
      fileList: [res.fileID],
      success: urlRes => {
        const permanentUrl = urlRes.fileList[0].tempFileURL;
        // 永久URL，用于写入NFC和分享
        console.log('永久URL:', permanentUrl);
      }
    });
  }
});
```

### 4.2 H5播放页技术实现

**URL格式设计**：
```
https://[云托管域名]/play.html?url=[录音URL]
```

**H5播放页核心代码**：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>录音播放</title>
  <style>
    /* 像素风样式 - 参考UI设计规范 */
    body {
      font-family: 'Courier New', monospace;
      background-color: #F4EDD6;
    }
  </style>
</head>
<body>
  <div class="player-container">
    <div class="icon">🔊</div>
    <audio id="audioPlayer" controls>
      您的浏览器不支持音频播放
    </audio>
    <p class="tip">碰一碰 NFC 播放录音</p>
  </div>

  <script>
    // 获取URL参数中的录音地址
    const urlParams = new URLSearchParams(window.location.search);
    const audioUrl = urlParams.get('url');

    if (audioUrl) {
      const audio = document.getElementById('audioPlayer');
      audio.src = decodeURIComponent(audioUrl);
      audio.play().catch(err => {
        console.log('自动播放受限:', err);
      });
    }
  </script>
</body>
</html>
```

### 4.3 NFC写入

**复用现有能力**：
- 使用现有的 `scan-dialog` 组件
- 将H5播放页URL作为"网页写入"类型写入NFC

**写入数据格式**：

```javascript
// 写入记录
{
  tnf: 1,           // URI Record
  type: 'U',        // URI type
  payload: h5PlayUrl // 完整H5页面URL
}
```

### 4.4 云托管部署

**部署步骤**：
1. 在微信开发者工具中上传H5播放页到云托管
2. 配置静态网站托管
3. 获取访问域名
4. 测试外部浏览器访问

**域名配置**：
- 使用云托管提供的默认域名（`*.tcloudbaseapp.com`）
- 或绑定自定义域名（需ICP备案）

## 五、异常处理

| 场景 | 处理方式 |
|------|----------|
| 录音权限被拒绝 | 弹窗提示"请允许录音权限"，提供跳转设置按钮 |
| 云存储上传失败 | Toast提示"上传失败，请重试"，保留录音供重新上传 |
| NFC写入失败 | 显示错误原因，提供重试按钮 |
| H5页面加载失败 | 显示友好错误提示"页面加载失败" |
| 录音URL无效 | 显示错误提示"录音文件不存在或已过期" |
| 手机不支持NFC | 隐藏NFC写入按钮，显示"您的设备不支持NFC" |

## 七、验收标准

### 7.1 功能验收

| 编号 | 验收项 | 检查方式 |
|------|--------|----------|
| R01 | 用户可以录制音频 | 手动测试 |
| R02 | 录音时长限制在2分钟内 | 手动测试 |
| R03 | 录音完成后可以试听 | 手动测试 |
| R04 | 可以重新录制 | 手动测试 |
| R05 | 录音可以上传到云存储 | 手动测试 |
| R06 | 上传后获得永久访问URL | 检查控制台输出 |
| R07 | 可以将URL写入NFC标签 | 需NFC设备测试 |
| R08 | 其他手机碰NFC可以打开H5页面 | 需另一台设备测试 |
| R09 | H5页面可以播放录音 | 手动测试 |

### 7.2 性能验收

| 编号 | 验收项 | 标准 |
|------|--------|------|
| P01 | H5页面加载时间 | < 3秒（4G网络） |
| P02 | 录音上传时间 | < 10秒（5分钟录音，4G网络） |
| P03 | NFC写入时间 | < 2秒 |

|------|--------|----------|
| C01 | iOS Safari浏览器 | 手动测试 |
