## 一、功能概述

### 1.1 需求背景

在 `NFC卡片助手` 现有的“应用写入 / 音乐写入 / 网页写入”能力之外，新增一条“本地音频上传后写入 NFC”的能力链路。

这项功能适用于：

- 个人语音留言
- 语音名片
- 商品语音说明
- 场馆语音介绍
- 活动音频指引

### 1.2 当前方案定位

当前代码实现的是：

- 用户从微信会话中选择本地音频文件
- 上传到微信云开发云存储
- 通过云函数写入数据库记录
- 返回可访问的播放链接
- 将该链接写入 NFC 标签

当前**不是**：

- 小程序内录音
- 独立 H5 播放页
- 视频上传方案

## 二、功能需求

### 2.1 功能清单

| 编号 | 功能名称 | 功能描述 | 优先级 | 备注 |
|------|----------|----------|--------|------|
| A01 | 音频文件选择 | 用户从微信会话中选择本地音频文件 | P0 | 使用 `wx.chooseMessageFile` |
| A02 | 音频文件校验 | 校验格式和大小是否合法 | P0 | 支持 `mp3 / m4a / wav`，大小上限 `20MB` |
| A03 | 云存储上传 | 音频文件上传到云存储 `audio-files/` 目录 | P0 | 依赖云开发 |
| A04 | 云函数建档 | 通过云函数写入音频记录到数据库 | P0 | 云函数名 `audio-file-service` |
| A05 | 链接写入 NFC | 将音频可访问链接写入 NFC 标签 | P0 | 复用 `scan-dialog` |
| A06 | 我的文件列表 | 展示已上传音频记录 | P0 | 新增 tab 页 |
| A07 | 去写入 | 从“我的文件”带回音频记录继续写卡 | P0 | 通过全局临时记录实现 |
| A08 | 删除文件记录 | 删除数据库记录并清理云端文件 | P0 | 删除后旧链接失效 |

### 2.2 用户流程

#### 2.2.1 上传并写入

```text
用户进入 NFC 首页
    ->
点击“本地音频”
    ->
从微信会话选择音频文件
    ->
校验格式与大小
    ->
上传到云存储
    ->
调用云函数写入数据库记录
    ->
返回可写入链接
    ->
打开 NFC 写卡弹窗
    ->
贴近 NFC 标签
    ->
写入成功
    ->
用户可选择“继续写入”或“回我的文件”
```

#### 2.2.2 我的文件复用

```text
用户点击底部“我的文件”
    ->
查看已上传音频列表
    ->
点击“去写入”
    ->
跳转回“写入本地音频”
    ->
页面显示“已带入文件”
    ->
继续写入 NFC
```

#### 2.2.3 删除流程

```text
用户在“我的文件”点击“删除”
    ->
弹出确认提示
    ->
提示删除后旧 NFC 链接失效
    ->
用户确认删除
    ->
删除记录并更新列表
```

## 三、页面与交互

### 3.1 页面清单

| 页面名称 | 页面路径 | 功能说明 |
|----------|----------|----------|
| 本地音频写入页 | `/pages/write-local-audio/write-local-audio` | 选择音频、上传、生成链接、写入 NFC |
| 我的文件 | `/pages/my-files/my-files` | 查看、复用、删除已上传音频 |
| 自定义 tabBar | `/custom-tab-bar` | 底部导航 `NFC / 我的文件` |

### 3.2 写入本地音频页面要求

- 页面标题为：`写入本地音频`
- 右上角不显示 Home 按钮
- 页面需要展示格式和大小限制说明
- 页面说明文案需明确：上传后会先写入云存储和数据库；若要保证写入 NFC 的链接长期有效，需要将 `audio-files/` 路径设置为公有读
- 页面主按钮文案会随状态变化
- 页面字段当前显示为“播放页链接”，但当前生成的是云存储可访问的播放链接，不是独立 H5 页面
- 当前需要覆盖这些页面状态：
  - 初始说明态
  - 已选择文件态
  - 上传中
  - 生成链接中
  - 已带入文件态
  - 错误提示态
- 错误提示态需覆盖格式不支持、文件过大、上传失败、链接不可用等 `inline-error` 场景

### 3.3 我的文件页面要求

- 页面标题为：`我的文件`
- 通过底部自定义 `tabBar` 进入
- 页面进入后先展示“正在加载我的文件”状态，再切换为空态或列表态
- 顶部展示统计信息：
  - 文件总数
  - 云端占用
- 列表项展示：
  - 文件名
  - 上传时间
  - 文件大小
  - 播放链接
- 当前默认最多展示最近 `50` 条有效记录
- 每条记录支持：
  - `去写入`
  - `删除`
- 若记录未获取到可用播放链接，则“去写入”按钮禁用，并提示检查云存储 `audio-files` 路径权限

### 3.4 删除确认要求

- 删除前必须弹出确认弹窗
- 弹窗需明确提示：

```text
删除后，已经写入 NFC 的旧链接将失效
```

### 3.5 NFC 成功态要求

本地音频写卡成功后，成功弹窗需要提供两个动作：

- `继续写入`
- `回我的文件`

## 四、数据与云端要求

### 4.1 云环境

- 云环境 ID：`cloud1-5gsbvhr91440e872`

### 4.2 云函数

- 云函数名称：`audio-file-service`
- 当前支持动作：
  - `create`
  - `list`
  - `delete`
- `create` 当前会校验 `fileId`、`fileName`、`fileSize`，并将 `fileExt` 一并写入数据库
- `list` 会为当前有效记录重新获取最新 `playUrl`；若获取失败，则返回空链接，由前端禁用“去写入”

### 4.3 数据库

- 集合名称：`audio_files`

当前记录字段包括：

- `openid`
- `fileId`
- `fileName`
- `fileExt`
- `fileSize`
- `playUrl`
- `status`
- `createdAt`
- `updatedAt`
- `deletedAt`

### 4.4 云存储

- 上传目录：`audio-files/`

说明：

- 当前方案将链接直接写入 NFC
- 为保证链接长期可用，`audio-files/` 路径需要配置为`公有读`
- “我的文件”页的链接可用性与“去写入”按钮状态，也依赖这一云存储权限配置

## 五、异常处理

| 场景 | 处理方式 |
|------|----------|
| 音频格式不支持 | 提示“仅支持 mp3 / m4a / wav 文件” |
| 音频文件过大 | 提示“文件超过 20MB，请压缩后重新上传” |
| 文件读取失败 | 提示“无法读取文件，请重新选择” |
| 上传失败 | 提示“上传失败，请稍后重试” |
| 链接不可用 | 提示检查云存储 `audio-files` 路径权限 |
| 删除失败 | 提示“删除失败，请稍后重试” |
| NFC 写入失败 | 弹窗提示失败原因并支持重试 |
