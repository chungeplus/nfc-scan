# NFC Scan 发布前测试清单

## 1. 测试目标

- 验证 `写入应用`、`写入音乐`、`写入网页` 三条主流程可正常完成。
- 验证 `NFC 扫描弹窗` 在成功、失败、重试、取消场景下状态正确。
- 验证 `开发者承诺` 弹窗仅首次进入首页时展示。
- 验证 `版本更新检测` 已在全局接入。
- 验证 `真实代码`、`UI图`、`原型图` 的关键交互是否一致。
- 在发布前识别阻塞上线的 `P0/P1` 问题。

## 2. 测试环境

- 平台：微信开发者工具、Android 真机、iOS 真机
- 系统：
  - Android 10 及以上，且设备支持 NFC
  - iOS 16 及以上，用于页面和交互验证
- 网络：
  - 正常网络
  - 弱网
  - 断网
- 标签：
  - 可写 NDEF 标签
  - 非 NDEF 标签
  - 已写入标签

## 3. 发布阻塞项

- `P0` 音乐页网络请求失败后可能卡在“解析中”，按钮无法恢复。`状态：已完成代码回归，待真机验证`
- `P0` QQ 音乐成功分支没有重置 `parsingLoading`，状态可能残留。`状态：已完成代码回归，待真机验证`
- `P0` 音乐链接解析到空结果时，仍可能继续写入无效 URI。`状态：已完成代码回归，待真机验证`
- `P0` NFC 弹窗点击“重试”时存在状态竞争，旧定时器可能覆盖新状态。`状态：已完成代码回归，待真机验证`
- `P0` NFC 弹窗重复发现同一标签时可能重复写入。`状态：已完成代码回归，待真机验证`
- `P1` 网页链接校验规则偏窄，带 `query/hash/port` 的合法 URL 可能被误判。`状态：已完成代码回归，待真机验证`
- `P1` 应用页 iOS 按钮可点，但提交时又提示未接入，交互一致性不足。`状态：已完成代码回归，待真机验证`
- `P1` 开发者承诺弹窗每次进入首页都会重复出现。`状态：已完成代码回归，待真机验证`
- `P1` 小程序未接入全局版本更新检测。`状态：已完成代码回归，待真机验证`

## 4. 冒烟测试

### 4.1 首页入口

- 打开首页，确认三个入口卡片可见。
- 首次进入首页时应展示 `开发者承诺` 弹窗。
- 点击 `了解` 后关闭弹窗，再次进入首页不再重复弹出。
- 点击 `写入应用`，可进入应用写入页。
- 点击 `写入音乐`，可进入音乐写入页。
- 点击 `写入网页`，可进入网页写入页。
- 不支持 NFC 的设备上，入口点击后应有明确提示。

### 4.2 应用写入

- 默认平台为 `Android`。
- 默认应用类型和包名正常回显。
- 点击 `应用类型` 可弹出选择应用弹框。
- Android 下弹框可正常选择分类和应用。
- iOS 下点击 `应用类型`，应和 Android 一样弹框。
- 选择后 `应用类型` 与 `应用包名` 正常回填。
- 自定义分类下可手动输入包名。
- 非法包名显示错误提示。
- 合法包名时 `写入 NFC 标签` 按钮可点击。
- 点击写入后弹出 NFC 扫描弹窗。

### 4.3 音乐写入

- 输入框为多行输入，文字从左到右、从上到下正常显示。
- 粘贴按钮可写入剪贴板内容。
- 网易云短链可识别。
- QQ 音乐分享链接可识别。
- 不支持的链接应提示 `链接不支持`。
- 网络失败时页面应从 `解析中` 恢复。
- 成功解析后可弹出 NFC 扫描弹窗。

### 4.4 网页写入

- 输入框为多行输入，文字从左到右、从上到下正常显示。
- 粘贴按钮可写入剪贴板内容。
- 不带协议的域名可自动补全 `https://`。
- 合法 URL 可进入 NFC 扫描弹窗。
- 非法 URL 显示错误提示。
- 验证带参数、锚点、端口的 URL 是否被正确识别。

### 4.5 版本更新

- 小程序启动后可正常注册版本更新检测逻辑。
- 有新版本时应弹出更新确认弹窗。
- 用户点击确认后应执行重启更新。
- 更新下载失败时应给出明确提示。

### 4.6 NFC 弹窗

- 打开时默认处于 `等待贴近标签` 状态。
- 发现标签后进入 `写入中`。
- 写入成功后展示成功态。
- 写入失败后展示失败态。
- 点击关闭可退出弹窗。
- 点击重试后可重新开始扫描。
- 快速连续打开/关闭弹窗，不应出现状态错乱。
- 非 NDEF 标签应有明确错误提示。

## 6. 回归重点

- 应用页平台切换后：
  - `应用类型`
  - `应用包名`
  - `按钮可点击状态`
  - `错误提示`
- 音乐页解析后：
  - `writeStatus`
  - `parsingLoading`
  - `scanVisible`
  - `records`
- 网页页校验后：
  - `urlError`
  - 自动补协议
  - 非法链接拦截
- NFC 弹窗：
  - `waiting`
  - `writing`
  - `success`
  - `error`
  - `retry`
- 首页：
  - `developerPromiseDialogVisible`
  - 本地缓存标记是否生效
- 全局：
  - 版本更新检测注册
  - 更新成功 / 失败提示

## 7. 发布建议

- 修复全部 `P0` 后再提审。
- `P1` 至少需要人工验证通过，并确认不影响主流程。
- 当前仓库未接入自动化测试，建议后续补：
  - 页面逻辑单测
  - 链接解析单测
  - NFC 弹窗状态流转测试
