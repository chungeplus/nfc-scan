# NFC直达小程序 UI 设计规范稿

## 1. 设计目标

- 设计关键词：像素风、明亮、纸张感、复古游戏 UI、可爱但不幼稚
- 参考方向：不是霓虹暗黑，也不是纯扁平网页，而是“纸张底 + 彩色像素块 + 粗黑描边 + 硬阴影”的 8-bit 视觉
- 目标结果：用户一眼看出是像素风，同时仍适合工具类小程序，保证信息清晰、操作直接

## 2. 视觉原则

- 所有核心组件都必须有明显黑色描边，不能出现弱边界
- 所有阴影使用硬阴影，不使用模糊阴影
- 所有主视觉区域优先使用大色块，而不是靠渐变堆气氛
- 页面底色以暖纸色为主，外层场景色用浅绿色或复古环境色承托
- 重要页面必须有“模块层级”，不能是表单字段简单往下堆
- 图标必须是可辨识的像素图标，不能仅靠文字代替

## 3. 配色规范

### 3.1 基础色

| 名称 | 色值 | 用途 |
|---|---|---|
| 场景绿 | `#88AA67` | 页面外层背景、舞台底色 |
| 纸张底 | `#F4EDD6` | 页面主体背景 |
| 纸张辅底 | `#EFE5C6` | 输入框、底栏、浅色容器 |
| 主描边 | `#2D2D2D` | 全局边框、图标主线 |
| 次文字 | `#626262` | 辅助说明、次级文本 |

### 3.2 功能色

| 名称 | 色值 | 用途 |
|---|---|---|
| 橙色 | `#F0BF66` | 首页主卡、活动卡、推荐卡 |
| 紫色 | `#CEB0EF` | 音乐相关卡片、辅助功能块 |
| 青色 | `#9FD9DF` | 网页、设备、信息卡 |
| 绿色 | `#B8D98B` | 成功态、确认态、功能强化 |
| 深绿色 | `#6EA85E` | 主确认按钮、成功提示 |
| 红色 | `#E86A58` | 失败态、提醒态、警示按钮 |
| 深红 | `#BE4A3A` | 错误文字、错误强调 |
| 黄色 | `#EFD86C` | 标签、导航激活、说明按钮 |
| 蓝色 | `#6F87D4` | 网页写入、普通主按钮 |
| 深蓝 | `#4E63AD` | 强调文字、链接类说明 |

### 3.3 配色使用规则

- 首页主卡建议优先使用 `橙 / 紫 / 青`
- 页面主按钮只能从 `深绿 / 蓝 / 紫` 里选
- 错误反馈必须统一使用 `红 / 深红`
- 弹窗标题条优先使用 `黄`，错误弹窗可使用 `红`
- 同一页面主色不超过 3 种，避免过花

## 4. 字体规范

- 字体方向：像素感等宽字体
- 当前原型可用字体：`"Courier New", Courier, monospace`
- 后续正式实现建议接入可商用像素字体资源

### 4.1 字号层级

| 层级 | 建议字号 | 用途 |
|---|---|---|
| 页面主标题 | `34px / 36rpx` | 预览页总标题、重要视觉标题 |
| 页面标题 | `18px / 32rpx` | 顶部导航标题 |
| 模块标题 | `15px / 28rpx` | “今日推荐”、“写入应用”等 |
| 卡片标题 | `16px / 30rpx` | 首页功能卡标题 |
| 正文 | `12px / 24rpx` | 输入内容、常规说明 |
| 辅助文案 | `10px / 20rpx` | 描述、提示、注释信息 |

### 4.2 文字规则

- 重要标题可以加轻微硬阴影
- 正文信息以清晰优先，不要做花哨发光
- 辅助文字统一使用次文字色，不要过浅

## 5. 边框与阴影

### 5.1 边框规则

- 主容器边框：`4px solid #2D2D2D`
- 轻组件边框：`3px solid #2D2D2D`
- 手机外壳边框：`5px solid #2D2D2D`

### 5.2 阴影规则

- 统一使用硬阴影，不允许 `blur`
- 标准阴影：`4px 4px 0 rgba(0, 0, 0, 0.14~0.18)`
- 强卡片阴影：`5px 5px 0 rgba(0, 0, 0, 0.16)`
- 大容器阴影：`8px 8px 0 rgba(0, 0, 0, 0.18~0.2)`

### 5.3 禁止项

- 禁止毛玻璃
- 禁止柔和投影
- 禁止强渐变塑料感
- 禁止无边框浅色块直接悬浮

## 6. 场景与背景

### 6.1 页面外层背景

- 使用浅绿色复古场景底
- 可以叠加稀疏像素网格、颗粒点、轻微噪点
- 目的不是抢视觉，而是让页面脱离普通白底网页感

### 6.2 页面内容背景

- 主背景统一为暖纸色
- 可叠加弱像素颗粒纹理
- 页面内容区不建议使用纯白

## 7. 页面结构规范

### 7.1 首页结构

- 顶部导航栏
- 推荐区标题
- 三张主功能卡
- 底部导航

### 7.2 表单页结构

- 顶部导航栏
- 功能标签区（如有）
- 输入模块区（输入框独占一行）
- 粘贴按钮区（按钮独占一行）
- 主操作按钮区

### 7.3 弹窗页结构

- 遮罩层
- 标题栏
- 内容区
- 操作区

## 8. 组件规范

### 8.1 顶部导航栏

- 高度：视觉上必须厚一点，不能太扁
- 结构：状态信息 + 标题行
- 风格：纯色大色块，不用透明
- 页面色建议：
- 首页：红色系
- 应用页：黄色系
- 音乐页：绿色系
- 网页页：蓝色系

### 8.2 首页功能卡

- 必须是大块彩色卡片
- 每张卡至少包含：像素图标、标题、按钮、小描述
- 三张卡必须在色相上区分明显
- 第一张卡作为主卡，视觉权重最高

### 8.3 按钮

#### 主按钮

- 高度：明显大于普通字段
- 边框：粗黑边
- 阴影：硬阴影
- 颜色：统一使用深绿色 `#6EA85E`，所有写入页保持一致
- 文案：字距略大，强化"像素按钮"感

#### 次按钮

- 用于粘贴、检测、解析、轻操作
- 可用黄色或浅纸底
- 宽度不需要太大，但必须有边框

### 8.4 输入框

- 背景：纸张辅底
- 边框：黑色粗边
- 必须有内压感，可通过 inset 阴影实现
- 禁止无边输入框

### 8.5 信息摘要卡（已废弃）

> 注：此组件已在最新版本中移除，不再使用。

- ~~用于展示平台、歌曲、写入内容摘要~~
- ~~背景比页面主体略亮~~
- ~~边框与阴影弱于主卡，但不能没有层级~~

### 8.6 列表卡

- 用于网页写入页、结果区、说明区
- 左侧建议带像素图标
- 右侧可有 `+ / > / 状态值`

### 8.7 底部导航

- 结构：图标 + 文案
- 激活项：黄色底 + 黑边 + 阴影
- 非激活项：纸张辅底
- 图标必须使用像素图标，不用常规线性图标

### 8.8 弹窗

- 必须有明显遮罩
- 标题栏和内容区要分层
- 普通弹窗标题优先黄色
- 错误弹窗标题可改红色
- 弹窗按钮不宜超过两个

### 8.9 Toast

- 尺寸要比原生 Toast 更有存在感
- 成功：绿色
- 警告：黄色
- 失败：红色
- 位置：页面中部偏上或正中

## 9. 图标规范

### 9.1 图标原则

- 所有核心图标都采用像素图标
- 图标首先要“能识别”，其次才是装饰
- 不接受“只是一个色块看起来像图标”

### 9.2 第一批必须精修的图标

- `home`
- `app`
- `music`
- `web`
- `nfc`
- `back`
- `user`
- `heart`
- `error`
- `success`

### 9.3 图标使用规则

- 首页大卡图标允许更大
- 表单/列表图标使用中号规格
- 底栏图标统一尺寸
- 一个页面内图标风格必须统一，不可混用

## 10. 动效规范

- 动效必须克制，不能做成现代 App 那种丝滑过渡
- 建议使用短时、硬切、跳帧感动画
- Toast 可以轻微弹出
- Loading 可以做像素旋转块
- 禁止模糊位移动画

## 11. 当前原型存在的问题

### 11.1 已解决的问题

- 已经脱离普通网页感
- 已经具备明显的像素配色方向
- 已经补上手机框、卡片、底栏、弹窗的基本层级

### 11.2 仍需优化的问题

- 首页主卡片还可以更精致
- 图标辨识度还可以继续提升
- 卡片边缘还不够像素锯齿化
- 一些页面装饰密度仍略低
- 局部文案排版还偏“网页表单”，可以更像游戏 UI

## 12. 下一步落地顺序

### 第一阶段

- 先精修首页
- 输出更高保真的首页卡片样式
- 定义最终按钮、标题栏、底栏样式

### 第二阶段

- 将首页样式规范复制到写入应用页
- 完成输入框、摘要区、按钮区统一

### 第三阶段

- 同步到音乐页、网页页
- 同步到开发者承诺弹窗、NFC 弹窗、Toast

## 13. 验收标准

- 用户第一眼能判断这是像素风，而不是普通网页风
- 页面之间风格统一，不会出现“首页像像素风，内页像后台页”
- 图标具备辨识度，不依赖文字解释
- 卡片、按钮、弹窗都具备清晰层次
- 所有重要操作区域都足够醒目
